## 测试要求
1. 使用 Google Test (gtest) 框架
2. 为每个公共方法生成测试用例
3. 覆盖以下场景：
   - 正常路径（happy path）
   - 边界条件（0、负数、极值、空值）
   - 异常处理（std::exception, 自定义异常）
   - 边界值分析
   - 内存安全（指针、引用）

4. 测试命名规范：
   - `TEST_F(ClassName, MethodName_Scenario)`
   - 使用描述性名称，例如：`TEST_F(CalculatorTest, AddTwoPositiveNumbers_ReturnsCorrectSum)`

5. 使用 Google Test 断言：
   - `EXPECT_EQ`, `ASSERT_EQ` - 相等性
   - `EXPECT_NE`, `ASSERT_NE` - 不等性
   - `EXPECT_TRUE`, `ASSERT_TRUE` - 布尔值
   - `EXPECT_FALSE`, `ASSERT_FALSE` - 布尔值
   - `EXPECT_THROW`, `ASSERT_THROW` - 异常
   - `EXPECT_ANY_THROW`, `ASSERT_ANY_THROW` - 任意异常
   - `EXPECT_NEAR`, `ASSERT_NEAR` - 浮点数比较

6. 使用测试固件（Test Fixture）：
   ```cpp
   class ClassNameTest : public ::testing::Test {{
   protected:
       ClassNameTest() {{}}
       ~ClassNameTest() override {{}}

       void SetUp() override {{{{
           // 测试前初始化
       }}}}

       void TearDown() override {{{{
           // 测试后清理
       }}}}

       // 测试数据成员
       ClassName obj;
   }};
   ```

7. 如果是 QT 项目：
   - 初始化 QT 应用（QCoreApplication 或 QApplication）
   - 使用 QTest 相关宏（如果需要）
   - 测试信号槽（如果需要）

8. 包含必要的头文件和 mock（如果需要）
